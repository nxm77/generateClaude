## 三、架构分析

### 3.1 ai-switch 架构亮点

#### 3.1.1 配额管理系统

**三层额度体系**:
```
个人额度 (Personal Quota)
  ├─ 按用户分配
  ├─ 支持公司/部门级作用域
  └─ 滚动窗口 (4/12/24/168h)

项目额度 (Project Quota)
  ├─ 按项目分配
  ├─ 支持公司/部门/项目级作用域
  └─ 滚动窗口 (4/12/24/168h)

补充额度 (Supplement Quota)
  ├─ 紧急池
  ├─ 按用户发放
  └─ 有过期时间
```

**预扣-结算流程**:
```go
1. ReserveQuota()  // 预扣（幂等，并发安全）
   ├─ 创建 QuotaReservation (status=reserved)
   ├─ 唯一键: (UserID, RequestID)
   └─ 冲突时回读已有记录

2. FinalizeQuota() // 结算（多退少补）
   ├─ 事务内加锁
   ├─ 更新 status=finalized/failed
   └─ 触发配额告警（异步）
```

#### 3.1.2 模型转发适配器

**适配器接口**:
```go
type Adaptor interface {
    GetRequestURL(resource *ModelResource) string
    SetupRequestHeader(req *http.Request, resource *ModelResource)
    ConvertRequest(payload map[string]any) ([]byte, error)
    ParseResponse(body []byte, statusCode int) (*RelayResponse, error)
    ParseStreamChunk(line []byte) (*StreamChunk, error)
    GetProviderName() string
}
```

**已实现适配器**:
- OpenAI Adaptor
- Anthropic Adaptor
- Gemini Adaptor

**Token 分类支持**:
- InputTokens (输入)
- OutputTokens (输出)
- CacheCreationTokens (缓存创建)
- CacheReadTokens (缓存读取)

#### 3.1.3 响应缓存系统

**三层缓存架构**:
```
L1: 内存 LRU 缓存
  ├─ 容量: 可配置 (默认 1000 条)
  ├─ TTL: 可配置 (默认 3600s)
  └─ 最快响应

L2: SQLite 持久化
  ├─ 跨会话共享
  ├─ 容量: 无限制
  └─ 支持流式缓存重放

L3: 语义向量索引
  ├─ 相似度匹配
  ├─ 阈值: 0.95
  └─ 智能缓存命中
```

**防击穿机制**: singleflight 模式

### 4.2 techtradechain 项目

#### 问题

1. **证书依赖关系** (高优先级)
   - 现象: 所有服务必须使用同一批证书

   - 影响: 服务间无法通信

   - 7 个子项目需要独立构建

   - 构建时间较长


