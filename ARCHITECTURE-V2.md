## 深度分析流程对比

### v1.0 流程 (不稳定)

```
┌──────────────────────────────────────────────────────────┐
│                    v1.0 深度分析流程                       │
└──────────────────────────────────────────────────────────┘

1. 查找 skill 文件
   ├─ 路径 1: $env:USERPROFILE\.claude\skills\...
   ├─ 路径 2: .claude\skills\...
   └─ ❌ 经常找不到 → 失败

2. 检查 skill 是否存在
   └─ ❌ 不存在 → 直接退出

3. 调用 claude CLI (依赖 skill 机制)
   ├─ claude -p "请使用 project-deep-analyzer skill..."
   └─ ❌ skill 加载失败 → 分析失败

4. 检查报告文件
   └─ ❌ 未生成 → 失败

5. 显示统计信息
   └─ ❌ 数据未被后续使用

成功率: ~40%
```

### v2.0 流程 (稳定)

```
┌──────────────────────────────────────────────────────────┐
│                    v2.0 深度分析流程                       │
└──────────────────────────────────────────────────────────┘

1. 查找 skill 文件
   ├─ 路径 1: .claude\skills\project-deep-analyzer.md
   ├─ 路径 2: $env:USERPROFILE\.claude\skills\...
   └─ ✅ 找到任一路径即可

2. 读取 skill 文件内容
   ├─ $skillTemplate = Get-Content $skillPath -Raw
   └─ ✅ 直接读取,不依赖 skill 机制

3. 构建完整 prompt
   ├─ $prompt = "$skillTemplate`n---`n项目信息..."
   ├─ 保存到临时文件 (便于调试)
   └─ ✅ 完全控制 prompt 内容

4. 调用 claude CLI (不依赖 skill 机制)
   ├─ claude -p $prompt
   └─ ✅ 直接传递 prompt,不需要 skill 加载

5. 检查报告文件
   ├─ 存在 → 解析 JSON
   └─ 不存在 → ✅ 优雅降级到基础模式

6. 返回结构化数据
   ├─ return $analysisReport
   └─ ✅ 供后续文档生成使用

成功率: ~95%
```

## 错误处理流程

```
┌─────────────────────────────────────────────────────────────┐
│                    v2.0 错误处理                             │
└─────────────────────────────────────────────────────────────┘

深度分析执行
    │
    ├─ skill 文件不存在?
    │      ├─ YES → 显示警告
    │      │        └─ return null → 使用基础模式 ✅
    │      └─ NO → 继续
    │
    ├─ claude CLI 不可用?
    │      ├─ YES → 显示错误
    │      │        └─ return null → 使用基础模式 ✅
    │      └─ NO → 继续
    │
    ├─ 分析执行失败?
    │      ├─ YES → 捕获异常
    │      │        └─ return null → 使用基础模式 ✅
    │      └─ NO → 继续
    │
    ├─ 报告文件未生成?
    │      ├─ YES → 显示警告
    │      │        └─ return null → 使用基础模式 ✅
    │      └─ NO → 继续
    │
    └─ JSON 解析失败?
           ├─ YES → 显示错误
           │        └─ return null → 使用基础模式 ✅
           └─ NO → 返回分析数据

结果: 无论哪个环节失败,都能生成文档 ✅
```

## 性能优化点

```
┌─────────────────────────────────────────────────────────────┐
│                    性能优化策略                              │
└─────────────────────────────────────────────────────────────┘

1. 文件扫描优化
   ├─ 限制扫描深度: -Depth 3
   ├─ 限制文件数量: Select-Object -First 50
   ├─ 排除无关目录: node_modules, .git, dist
   └─ 效果: 扫描时间 < 10秒

2. 深度分析优化
   ├─ 保存 prompt 到临时文件 (避免重复构建)
   ├─ 缓存分析结果 (存储在全局变量)
   └─ 效果: 避免重复分析

3. 文档生成优化
   ├─ 条件判断: 有数据用详细版,无数据用基础版
   ├─ 避免重复扫描: 复用基础分析数据
   └─ 效果: 生成时间 < 5秒

总耗时:
- 基础模式: ~25秒
- 深度模式: ~3-5分钟 (主要是 Claude 分析时间)
```
